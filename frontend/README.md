## Docker (production)

This repository includes a `Dockerfile` that builds the Next.js app using `output: 'standalone'` and packages a minimal runtime image.

Build the image (run from the `frontend/` directory):

```bash
docker build -t jsfitness-frontend:prod .
```

Run the container:

```bash
docker run -e NODE_ENV=production -p 3000:3000 --name jsfitness-frontend jsfitness-frontend:prod
```

Notes:
- The standalone build outputs a self-contained server in `.next/standalone` which the Dockerfile uses.
- Pass any required environment variables (for example analytics keys) via `-e` or an env file.
- For production, serve behind a TLS-enabled reverse proxy.
